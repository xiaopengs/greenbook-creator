## 配文

**封面**：73%的 AI Skill 写太长反而帮倒忙。复旦×微软 5193 个 Skill 大规模评测：不是写得越多越好。

**P2**：维度1「可操作的具体性」精度 66.0%。不是说"注意规范"，而是精确到类名、方法名、配置项。模糊规则=废纸。

**P3**：维度2「失败机制编码」精度 65.5%。每条规则回答三问——什么触发？什么原因？什么症状？用 OkHttp 超时→流式中断→SocketTimeoutException 这种精确度去写。

**P4**：维度3「高风险黑名单」精度 64.6%。每条 Skill 必须有禁止项+危害+替代方案。没有黑名单=没有刹车。

**P5**：元规则：3-5 条核心规则 > 10+ 条全面规则。每多一条冗规，负迁移风险 +2.7%。少，就是多。

**P6**：检查清单——①有具体类名/方法名吗？②有触发+根因+症状吗？③有禁止项+危害+替代方案吗？收藏这篇，每次发布 Skill 前过一遍。
